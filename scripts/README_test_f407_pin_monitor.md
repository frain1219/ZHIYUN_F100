# F407引脚监听自动化测试工具

## 功能说明

这个Python脚本用于自动化测试STM32F407VGT6探针板的引脚监听功能，验证APM32F103CB到STM32F407V的接线是否正常（覆盖接线文档3.1-3.4模块）。

## 测试范围

- **3.1 系统状态与控制链路**: PE0, PE5, PE1, PE2
- **3.2 按键输入模块**: PE6, PC0
- **3.3 OLED抓取模块**: PE8, PE9, PE10, PE11, PE12
- **3.4 编码器与PWM模块**: PC1, PC2, PC3, PE7

## 安装依赖

```bash
pip install pyserial
```

## 使用方法

### 1. 基本使用（自动检测端口）

```bash
cd /Users/milocheung/Documents/codeProject/ZHIYUN_F100/scripts
python test_f407_pin_monitor.py --duration 10
```

### 2. 手动指定端口

```bash
python test_f407_pin_monitor.py --port /dev/tty.usbmodem123456 --duration 10
```

### 3. 列出所有可用串口

```bash
python test_f407_pin_monitor.py --list-ports
```

### 4. 保存测试报告

```bash
python test_f407_pin_monitor.py --output test_report.txt
```

## 参数说明

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| --port | -p | 串口设备路径 | 自动检测 |
| --baudrate | -b | 波特率 | 115200 |
| --duration | -d | 监听持续时间（秒） | 5.0 |
| --list-ports | -l | 列出所有可用串口 | - |
| --output | -o | 测试报告输出文件路径 | - |

## 测试流程

1. **连接测试**: 通过PING命令验证与F407的通信
2. **引脚列表获取**: 获取F407固件中定义的所有监听引脚
3. **监听测试**: 开启订阅模式，监听指定时长内的引脚状态变化事件
4. **引脚验证**: 验证测试范围内的所有引脚是否正常工作
5. **报告生成**: 生成详细的测试报告

## 测试建议

### 测试前准备

1. 确保F407已烧录正确的固件
2. 确保F103和F407已正确共地
3. 确保所有数字线已串联100欧电阻
4. 确保F103正常工作并有信号活动

### 测试时操作

在监听测试期间（默认5秒），建议在F103侧触发以下操作：

- **按键模块**: 按下/释放按键
- **OLED模块**: 触发OLED刷新（如果有显示内容变化）
- **编码器模块**: 转动编码器
- **PWM模块**: 触发PWM输出（如果有相关功能）

这样可以产生更多的信号变化事件，帮助验证监听功能。

## 故障排查

### 问题1: 找不到F407设备

**解决方案**:
1. 检查F407是否已通过USB连接到电脑
2. 检查驱动是否正确安装
3. 使用 `--list-ports` 查看所有可用端口
4. 手动指定端口使用 `--port` 参数

### 问题2: 连接失败

**解决方案**:
1. 确认F407固件已正确烧录
2. 确认USB CDC功能正常
3. 尝试重新插拔USB
4. 检查是否有其他程序占用了串口

### 问题3: 引脚监听无事件

**解决方案**:
1. 检查接线是否正确（参考接线文档）
2. 确认F103侧对应信号是否正常工作
3. 检查100欧电阻是否正确串联
4. 延长监听时间 `--duration 20`

### 问题4: 部分引脚失败

**解决方案**:
1. 检查失败引脚的接线
2. 使用万用表验证信号是否到达F407引脚
3. 检查F407引脚是否损坏
4. 确认引脚配置是否正确

## 技术细节

### 串口协议

- **通信方式**: USB CDC虚拟串口
- **波特率**: 115200（USB CDC下实际不依赖波特率）
- **数据格式**: JSON
- **命令格式**: 文本命令 + 换行符

### 主要命令

- `PING`: 测试连接
- `LIST_PINS`: 获取引脚列表
- `GET_STATUS`: 获取当前状态快照
- `SUBSCRIBE ON/OFF`: 开启/关闭事件订阅
- `GET_STATS`: 获取统计信息

### 事件类型

- **POLL事件**: 轮询检测到的状态变化
- **EXTI事件**: 外部中断捕获的状态变化（更精确）
