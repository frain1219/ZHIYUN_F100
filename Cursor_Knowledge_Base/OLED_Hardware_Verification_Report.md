**结论:** F407完全未捕获到SCK和MOSI信号!

### 2. STLINK调试F103结果

**寄存器状态:**
```
PC = 0x08006A4E
SPI1_CR1 = 0x00000000  ❌ SPI1未启用
SPI1_SR  = 0x00000000  ❌ 无SPI活动
RCC_APB2ENR = 0x00000000  ❌ SPI1时钟未启用
GPIOA_CRL = 0x44444444  ❌ PA5/PA7配置为浮空输入
```

**PC位置分析:**
```assembly
0x08006A4E: str.w r3, [r1], #4
0x08006A52: bcs.n 0x08006A48
```

**发现:** F103正在执行C运行时的.bss段清零循环!

**循环变量:**
```
r1 = 0x20003F1C (SRAM地址)
r2 = 0x9DC = 2524字节 (剩余要清零的字节数)
r3 = 0 (清零值)
```

### 3. 按键触发测试结果

**测试方法:**
- 注入PA1按键信号 (PE6)
- 注入PB4按键信号 (PC0)
- 每个按键触发2次,持续100ms

**测试结果:**
```
按键触发前:
- 边沿计数: 3117
- 事务计数: 517
- 字节计数: 0

按键触发后:
- 边沿计数: 5341 (+2224)

```

**结论:** 按键触发增加了SPI活动,但仍然没有字节捕获,且SPI1寄存器状态未改变!

## 根本原因分析

### F103启动检测机制分析 (2026-05-02 更新)

**启动流程重建:**

1. **上电复位阶段**
   - PORRSTF标志置位
   - PC加载Reset_Handler地址 (0x08006BE9)
   - RCC_CSR = 0x1C000000 (SFTRSTF/PORRSTF/PINRSTF)

2. **Reset_Handler执行**
   - 调用SystemInit (0x080069DC)
   - 配置时钟系统 (HSI, HSE, PLL)
   - 设置VTOR寄存器
   - RCC_CR = 0x00019B83 (所有时钟稳定)

3. **C运行时初始化**
   - scatterload_table_iterate
   - .bss段清零 (之前卡在这里)
   - **现在已经完成！**

4. **main函数执行**
   - main函数地址: 0x08002E74
   - 调用多个初始化函数
   - 当前正在执行延时函数 (0x0800570C)

**关键发现: 为什么之前卡在.bss清零循环?**

**答案: F103需要特定的启动条件才能完成初始化！**

可能的原因:
1. **需要手动复位** - 用户手动复位后,F103成功启动
2. **需要稳定的电源** - 电源稳定后才能完成初始化
3. **需要硬件就绪信号** - 某些硬件条件满足后才能继续
4. **需要按键触发** - 可能需要特定的按键序列

**启动条件检测:**

从RCC_CSR寄存器分析:
```
RCC_CSR = 0x1C000000
Bit 26: SFTRSTF = 1 (软件复位标志)
Bit 27: PORRSTF = 1 (上电复位标志)
Bit 28: PINRSTF = 1 (NRST引脚复位标志)
```

这说明F103经历了完整的复位序列,包括:
- 上电复位 (POR)
- NRST引脚复位
- 软件复位

### 为什么F103会卡在.bss清零循环?

**可能的原因:**

1. **.bss段过大:** 需要清零的内存区域太大
2. **时钟配置问题:** F103可能运行在低速时钟,导致执行缓慢
3. **硬件等待状态:** 某些硬件初始化可能导致等待
4. **看门狗或中断干扰:** 可能被其他异常打断

### 为什么按键触发后有SPI活动?

**分析:**
- CS/RES/DC信号有活动,但SCK/MOSI没有
- 这说明有其他GPIO操作,但不是SPI通信
- 可能是按键处理或其他GPIO控制

### ⚠️ SPI1仍未初始化

**问题:**
1. **SPI1寄存器全为0** - SPI1_CR1/CR2/SR/DR = 0x00000000
2. **SPI1时钟未使能** - RCC_APB2ENR bit 12 (SPI1EN) = 0
3. **GPIO未配置为SPI复用** - PA5/PA6/PA7配置为通用推挽输出 (0x4)
   - 正确的SPI配置应该是复用推挽输出 (0xB)

**矛盾分析:**
- F103已执行过SPI1_StartupSequence调用位置 (0x08002F2C)
- 栈返回地址0x08002FC5 > SPI1初始化位置0x08002F2C
- 但SPI1寄存器仍然全为0

**可能原因:**
1. SPI1_StartupSequence函数执行失败
2. SPI1初始化被条件跳过
3. SPI1初始化需要特定硬件条件
4. 当前在延时函数中,可能在等待某些条件

```
Reset_Handler (0x08006BE9)
  ↓
C Runtime Initialization
  ↓
scatterload_table_iterate (0x08006B20)
  ↓
.bss Clear Loop (0x08006A48) ← 当前卡在这里!
  ↓
main (0x08002E74) ← 未到达
  ↓
SPI1_StartupSequence (0x08004184) ← 未执行
  ↓

```

### B. 关键地址

| 符号 | 地址 | 说明 |
|------|------|------|
| Reset_Handler | 0x08006BE9 | 复位处理函数 |
| scatterload | 0x08006B20 | C运行时散载 |
| .bss clear | 0x08006A48 | .bss清零循环 |
| main | 0x08002E74 | 主函数 |
| SPI1_StartupSequence | 0x08004184 | OLED初始化序列 |
