# 智云 F100 棒灯逆向工程报告

**文档状态：** 已完成拓扑逆向与初步测量
**设备对象：** 智云 F100 棒灯（支持最高 100W 双色温/RGB 输出）
**核心构成：** 主板（充放电、电源路径管理、主控）、副板（降压/升压驱动、混色）

---

## 1. 核心元器件清单
* **主控 MCU：** APM32F103C8（负责逻辑控制与电源路径切换）
* **充放电控制器：** PL5500（固定配置为**充电模式**，OTG引脚接地）
* **LED 驱动（副板）：** 2x MPS MP3398E（4 通道 WLED 升压控制器，输入 4.5-33V，输出耐压 80V） PWM 控制
* **电池：** 6 串 18650 电池组（无 NTC 测温设计），满电约 25.2V。
* **散热风扇：** 2针接口散热风扇（仅VCC/GND，不支持标准4-pin PWM控速，无转速反馈）（新修正）
* **HUSB238：** 用于 Type-C PD 输入，支持最大 100W 输出
* **升威EC18中空旋转编码器：** 用于调整输出功率、色温、RGB 颜色等

## 2. 动态电源路径拓扑 (DPPM) 逆向
系统电源架构基于复杂的 Back-to-Back (B2B) MOS 管矩阵进行动态路由。存在两个核心能量节点：
* **节点 `VS` (系统母线)：** 连接至副板接口 Pin 1/2/3/4，直接为 MP3398E 供电以点亮 LED。
* **节点 `R033` (PL5500 输入端)：** PL5500 Side 2 采样电阻（0.033Ω）所在节点，是外部供电与充电电路的十字路口。

### 供电路径详情：
1.  **DC 24V 输入模式：** * DC 24V -> B2B MOS -> 汇入 `VS` 节点（直接点灯）。
    * 同时，`VS` 节点 -> B2B MOS -> `R033` 节点 -> PL5500（降压）-> 电池（充电）。
2.  **Type-C PD 输入模式（根据PD功率动态分配）：**
    * Type-C (副板 Pin 9/10/11) -> B2B MOS -> 汇入 `R033` 节点。
    * **PD功率 &lt; 50W 时：**
        * `R033` -> PL5500（升/降压）-> `R040` (0.040Ω) -> 电池（仅充电，不给棒灯供电）。
        * 可开启全功率（提示：充电速率低于耗电速率，无法长时间边充边用）。
    * **50W &lt; PD功率 &lt; 100W 时：**
        * 在 `R033` 节点分流：
            * 支路 A：`R033` -> B2B MOS -> 汇入 `VS` 节点（优先给棒灯供电）。
            * 支路 B：`R033` -> PL5500（升/降压）-> `R040` (0.040Ω) -> 电池（剩余功率给电池充电）。
        * 不可开启全功率（屏幕提示功率不足）。
    * **PD功率 &gt; 100W 时：**
        * 在 `R033` 节点分流：
            * 支路 A：`R033` -> B2B MOS -> 汇入 `VS` 节点（优先给棒灯供电）。
            * 支路 B：`R033` -> PL5500（升/降压）-> `R040` (0.040Ω) -> 电池（剩余功率给电池充电）。
        * 可开启全功率。
3.  **单电池放电模式：**
    * 电池 -> 专用 B2B MOS -> 汇入 `VS` 节点（点灯）。（*注：PL5500 不参与放电*）
4.  **100W 满血输出模式 (Type-C + 电池协同供电)：**
    * Type-C 电能流向 `VS` 节点。
    * 电池电能直接通过 B2B MOS 流向 `VS` 节点。
    * 两者在 `VS` 节点物理并联汇流，共同提供 100W 功率。此时 PL5500 处于关闭状态。

*(注：系统设定在插入 Type-C 进行电源切换时，LED 会强制熄灭 1-2 秒，执行先断后通逻辑。)*

---
### PD输入策略
1. PD功率<50W时，PD仅给电池充电，可开启全功率(提示:充电速率低于耗电速率,无法长时间边充边用)
2. 50W< PD功率 <100W时，优先给棒灯供电,剩余功率给电池充电,不可开启全功率（屏幕提示功率不足）
3. PD功率>100W时,优先给棒灯供电,剩余功率给电池充电,可开启全功率;
