# HUSB238 规格书 (固件逆向版)

> **文档说明**: 本文档仅保留对APM32F103CBT6固件逆向有用的技术信息,重点突I2C通信接口和PD电源管理功能。HUSB238是Type-C PD受电端控制器,通过I2C与MCU通信,是电源管理策略的核心。

---

## 一、芯片概述

**型号**: HUSB238  
**类型**: USB Power Delivery (PD) 受电端控制器 (PD Sink)  
**封装**: DFN-10L (3mm x 3mm)

### 核心特性
- 支持USB PD3.0和Type-C V1.4标准
- 支持高达100W快充功率
- **通用I2C通信接口** (关键!)
- 支持Apple divider 3、BC1.2 SDP/CDP/DCP充电协议
- 工作电压范围: 3V ~ 25V

## 三、I2C通信接口 (逆向核心)

### 3.1 I2C基本参数

| 参数 | 数值 | 说明 |
|------|------|------|
| **I2C地址** | **0x08** | ⚠️ **关键信息** |
| I2C角色 | 从机 | MCU为主机 |
| 电压范围 | 2.5V ~ 5.5V | 兼容3.3V MCU |
| 低电平输入电压 | < 0.4V | - |
| 高电平输入电压 | > 1.4V | - |
| 输出低电平电压 | < 0.4V | @2mA负载 |

### 3.2 I2C工作流程

```
1. POR(上电复位)后,HUSB238从PD电源适配器接收充电器功率信息
2. HUSB238将充电器功率信息保存在寄存器中
3. MCU通过I2C总线访问HUSB238寄存器
4. MCU选择合适的PDO来请求外部合适的电压档位
```

### 3.3 I2C优先级

⚠️ **重要**: I2C具有最高优先级
- 如果使用I2C选择请求PDO,它会取代由VSET、ISET引脚和出厂设置的内部RDO
- 在写入I2C命令后,HUSB238将会请求I2C所选的源PDO

## 四、PDO选择逻辑 (逆向关键)

### 4.1 RDO测定方法

HUSB238通过两种方法确定请求的RDO(请求数据对象):

1. **由VSET和ISET引脚设置** (硬件配置)
2. **由I2C命令设置** (MCU控制)

**HUSB238比较两个值,并使用较低的值作为其RDO。**

### 4.2 PDO选择流程

在确定RDO后,HUSB238从最高电压循环通过PD充电器提供的PDO,找到第一个满足以下条件的PDO:

1. `SOURCE_PDO_VOLTAGE ≤ RDO_VOLTAGE`
2. `SOURCE_PDO_CURRENT ≥ RDO_CURRENT`

**如果满足条件**:
- HUSB238会请求该PDO
- 将运行电流设置为RDO的电流值

**如果不满足条件**:
- HUSB238将继续比较第二高电压源PDO
- 或直接请求5V的源PDO(默认行为)

### 4.3 VSET引脚配置 (硬件)

VSET引脚上采用固定的100μA电流源输出。

| RVSET (kΩ) | VSET_VOLTAGE (V) |
|------------|------------------|
| 0 | 5 |
| 6.04 | 9 |
| 10 | 12 |
| 14 | 15 |
| 17.8 | 18 |
| Open | 20 |

**推测**: 智云F100可能使用硬件电阻配置默认请求20V电压。

### 4.4 ISET引脚配置 (硬件)

ISET引脚上采用固定的100μA电流源。

| RISET (kΩ) | ISET_CURRENT (A) |
|------------|------------------|
| 0 | 1.25 |
| 4.53 | 1.5 |
| 7.5 | 1.75 |
| 10.5 | 2 |
| 13.7 | 2.25 |
| 16.5 | 2.5 |
| 19.6 | 2.75 |
| 22.6 | 3 |
| Open | 3.25 |

**推测**: 智云F100可能使用硬件电阻配置默认请求最大电流。

## 五、推测I2C寄存器映射

⚠️ **注意**: 文档未提供详细的寄存器映射,以下为基于功能的推测,需要逆向验证。

### 5.1 推测寄存器列表

| 寄存器地址(推测) | 寄存器名称 | 功能 | 读/写 |
|-----------------|-----------|------|------|
| 0x00 - 0x0F | PDO_INFO_REG | 存储PD电源适配器提供的PDO信息 | R |
| 0x10 | PDO_SELECT_REG | 选择请求的PDO编号 | R/W |
| 0x11 | STATUS_REG | 连接状态、保护状态等 | R |
| 0x12 - 0x13 | VBUS_VOLTAGE_REG | 当前VBUS电压(ADC值) | R |
| 0x14 - 0x15 | VBUS_CURRENT_REG | 当前VBUS电流(ADC值) | R |
| 0x16 | RDO_VOLTAGE_REG | 请求的电压值 | R/W |
| 0x17 | RDO_CURRENT_REG | 请求的电流值 | R/W |

### 5.2 推测PDO信息格式

每个PDO可能占用4字节,包含:
- 电压(单位: 50mV)
- 电流(单位: 10mA)
- PDO类型(固定/可变/电池等)

## 七、保护机制

### 7.1 过压保护 (OVP)

| 参数 | 数值 | 说明 |
|------|------|------|
| OVP阈值 | 最大请求电压的1.2倍 | - |
| Debounce时间 | 50μs | - |

**保护动作**:
1. 关闭外部PMOS
2. 进入放电状态
3. 等待重新连接

### 7.2 欠压保护 (UVP)

| 参数 | 数值 | 说明 |
|------|------|------|
| UVP阈值 | 请求电压减2V | 默认禁用 |
| Debounce时间 | 1ms | - |

**注意**: UVP功能默认未启用。

### 7.3 过温保护 (OTP)

| 参数 | 数值 | 说明 |
|------|------|------|
| OTP阈值(上升) | 150°C | - |
| OTP阈值(下降) | 130°C | - |

**保护动作**:
- 结温超过150°C时,请求5V PDO降低功率
- 温度降至130°C后,重新协商之前的PDO

## 八、关键时序参数

| 参数 | 数值 | 说明 |
|------|------|------|
| PD协商超时 | 1.5s | 超时后切换到传统充电器检测 |
| OVP debounce | 50μs | 过压保护延迟 |
| UVP debounce | 1ms | 欠压保护延迟 |
| VSET/ISET debounce | 2ms | 电压/电流设置延迟 |

## 九、逆向推理总结

### 9.1 MCU与HUSB238的交互

| 功能 | 接口 | 推测MCU连接 | 说明 |
|------|------|-----------|------|
| I2C通信 | SDA/SCL | I2C1_SDA/I2C1_SCL | 主要通信接口 |
| PDO信息读取 | I2C | 地址0x08 | 读取PD电源适配器能力 |
| PDO选择 | I2C | 地址0x08 | 选择请求的电压/电流 |
| VBUS监测 | I2C | 地址0x08 | 读取当前电压/电流 |
| 连接状态 | I2C | 地址0x08 | 读取连接状态寄存器 |

### 9.2 维修手册关联

根据维修手册的PD输入策略:
- MCU需要知道PD功率来决定供电策略
- PD功率 < 50W: 仅充电
- 50W < PD功率 < 100W: 优先供电,剩余充电
- PD功率 > 100W: 可全功率输出

推测MCU通过I2C读取PDO信息,计算PD功率,然后执行相应的电源管理策略。

### 9.3 固件逆向重点

1. **查找I2C初始化代码**:
   - 搜索I2C1配置
   - 查找GPIO复用配置(PB6/PB7或PB10/PB11)

2. **查找HUSB238通信函数**:
   - 搜索I2C地址0x08
   - 查找I2C读写函数

3. **查找PDO选择逻辑**:
   - 搜索PDO选择相关代码
   - 分析电压/电流请求算法

4. **查找电源管理策略**:
   - 搜索PD功率计算代码
   - 分析供电模式切换逻辑

### 9.4 不确定部分

- **不确定**: 具体的I2C寄存器映射(文档未提供)
- **不确定**: MCU是否监测VBUS电压/电流,还是仅依赖HUSB238
