# STM32F407VGT6 (LQFP100) 引脚重新分配计划

## 问题分析

用户反馈：STM32F407VGT6 (LQFP100封装) 中，部分PG和PF引脚未引出，导致无法使用。

### STM32F407VGT6 (LQFP100) 引脚可用性

根据STM32F407VGT6的封装规格：

**可用GPIO端口：**
- GPIOA: PA0-PA15（大部分可用，PA11/PA12用于USB）
- GPIOB: PB0-PB15（大部分可用）
- GPIOC: PC0-PC15（部分可用，PC14/PC15用于LSE）
- GPIOD: PD0-PD15（大部分可用，PD0/PD1用于HSE）
- GPIOE: PE0-PE15（大部分可用）
- GPIOF: **仅PF0-PF6可用**（ADC引脚）
- GPIOH: PH0-PH1（晶振引脚，可用作GPIO）

**不可用GPIO端口：**
- GPIOF: **PF7-PF15在LQFP100中未引出**
- GPIOG: **PG0-PG15在LQFP100中全部未引出**

### 当前配置中不可用的引脚

根据 `probe_pins.c` 和接线文档，以下引脚在LQFP100封装中不可用：

| 原引脚 | 功能模块 | 功能描述 | 状态 |
|--------|---------|---------|------|
| PG2 | 按键输入模块 | F103 PB4 右键输入 | ❌ 不可用 |
| PG5 | 编码器模块 | F103 PB5 MO_PWM | ❌ 不可用 |
| PG6 | 编码器模块 | F103 PB6 编码器A相 | ❌ 不可用 |
| PG7 | 编码器模块 | F103 PB7 编码器B相 | ❌ 不可用 |
| PF7 | 副板控制模块 | F103 PB12 AUX | ❌ 不可用 |
| PF8 | 副板控制模块 | F103 PB13 AUX/STAT | ❌ 不可用 |
| PF9 | 副板控制模块 | F103 PB14 EN测试点 | ❌ 不可用 |
| PF10 | 副板控制模块 | F103 PB15 ADIM测试点 | ❌ 不可用 |
| PF11 | 副板控制模块 | F103 PA8 W_PWM | ❌ 不可用 |
| PF12 | 副板控制模块 | F103 PA9 C_PWM | ❌ 不可用 |
| PF14 | 调试口监听模块 | F103 PA13 SWDIO | ❌ 不可用 |
| PF15 | 调试口监听模块 | F103 PA14 SWCLK | ❌ 不可用 |

## 引脚重新分配方案

### 可用引脚资源分析

**已使用引脚：**
- PE0-PE12（系统控制、按键、OLED模块）
- PF3-PF6（ADC模块）

**可用替代引脚：**
- PA10, PA15
- PB8, PB9, PB12, PB13, PB14, PB15
- PC0, PC1, PC2, PC3, PC4, PC5, PC6, PC7, PC8, PC9, PC10, PC11, PC12, PC13
- PD2, PD3, PD4, PD5, PD6, PD7, PD8, PD9, PD10, PD11, PD12, PD13, PD14, PD15
- PE13, PE14, PE15

### 推荐引脚重新映射方案

#### 方案一：优先使用GPIOC和GPIOD（推荐）

**优势：**
- GPIOC和GPIOD在LQFP100封装中引脚充足
- 避免与PA/PB的特殊功能冲突
- 便于PCB布线

**映射表：**

| 原引脚 | 新引脚 | 功能模块 | 功能描述 | 类型 |
|--------|--------|---------|---------|------|
| PG2 | PC0 | 按键输入模块 | F103 PB4 右键输入 | 数字监听/注入 |
| PG5 | PC1 | 编码器模块 | F103 PB5 MO_PWM | 数字监听（EXTI） |
| PG6 | PC2 | 编码器模块 | F103 PB6 编码器A相 | 数字监听（EXTI） |
| PG7 | PC3 | 编码器模块 | F103 PB7 编码器B相 | 数字监听（EXTI） |
| PF7 | PD2 | 副板控制模块 | F103 PB12 AUX | 数字监听/注入 |
| PF8 | PD3 | 副板控制模块 | F103 PB13 AUX/STAT | 数字监听/注入 |
| PF9 | PD4 | 副板控制模块 | F103 PB14 EN测试点 | 数字监听/注入 |
| PF10 | PD5 | 副板控制模块 | F103 PB15 ADIM测试点 | 数字监听/注入 |
| PF11 | PD6 | 副板控制模块 | F103 PA8 W_PWM | 数字监听/注入 |
| PF12 | PD7 | 副板控制模块 | F103 PA9 C_PWM | 数字监听/注入 |
| PF14 | PC4 | 调试口监听模块 | F103 PA13 SWDIO | 仅监听，禁止驱动 |
| PF15 | PC5 | 调试口监听模块 | F103 PA14 SWCLK | 仅监听，禁止驱动 |

#### 方案二：使用GPIOA和GPIOB（备选）

如果GPIOC/GPIOD有其他用途，可以使用GPIOA/GPIOB的空闲引脚：

| 原引脚 | 新引脚 | 功能模块 | 功能描述 |
|--------|--------|---------|---------|
| PG2 | PA10 | 按键输入模块 | F103 PB4 右键输入 |
| PG5 | PB8 | 编码器模块 | F103 PB5 MO_PWM |
| PG6 | PB9 | 编码器模块 | F103 PB6 编码器A相 |
| PG7 | PB12 | 编码器模块 | F103 PB7 编码器B相 |
| PF7-PF12 | PA15, PB13-PB15, PA10 | 副板控制模块 | - |
| PF14-PF15 | PC4-PC5 | 调试口监听模块 | - |

## 风险评估

### 潜在风险

1. **硬件兼容性：** 新引脚是否在目标板卡上实际可用
2. **中断冲突：** 新引脚的EXTI配置是否与其他功能冲突
3. **PCB布线：** 新引脚的物理位置是否便于接线

### 缓解措施

1. 在修改前确认目标板卡的引脚可用性
2. 检查STM32F407的EXTI线路分配，确保无冲突
3. 提供多种引脚映射方案供用户选择
